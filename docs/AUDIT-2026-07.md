## Verdict — right product, wrong discipline, fixable

The product instinct is correct and defensible: nothing on the market combines web + realtime-collab +
beat-grid-native + department cue-sheets + console/DAW exports, and separately nothing marries
waveform/count annotation with music-synced formation animation. cue.mith.studio stands on both
unoccupied intersections. Code is well-crafted at the line level.

Three things pull against it: (1) **existential ops gap** — no version control, no DB backups, entire
codebase on one VPS disk; (2) **focus problem** — two very different markets at once while the priority
list stalled at ~25%; (3) **product-vs-plumbing gap** — QLab-grade depth, no onboarding, no undo, dev
vocabulary throughout.

Scorecard: product/market-fit A− · code craft B+ · **operational maturity F** · UX maturity C− ·
strategic focus C · execution follow-through C.

## THE STRATEGIC FORK — pick one market to make excellent

Recommendation: **festival/live-production first, dance second, never both at 50/50.** Build is ~80%
festival-shaped; whitespace better-defended (export moat + rekordbox data moat); production companies
already pay $50–300/mo for waveform-less rundown tools. Keep the dance engine warm (formation work is
differentiated; shared core means it doesn't rot).

Counter-case for dance-first: lower quality bar to be best ("no A/S tier apps" per independent review),
clearer individual willingness-to-pay ($450–1,000/routine choreography fees), far lower reliability
stakes (a formation chart doesn't crash at 22:00; a web tool at FOH fights offline prejudice).

**Deciding criterion:** faster near-term revenue at lifestyle scale → dance-first. More defensible
long-term moat + appetite to grind conservative pro buyers → festival-first. Both defensible; doing
both is not.

## Market notes

- **Festival quadrant (web + collab + beat-native):** one funded-looking rival (time-line.io — free
  beta, no exports, no traction), one waitlist (EdiTour, "300 founding designers"). Desktop incumbents
  (CuePoints £128, ShowKontrol $1,999) own workflow trust; rundown SaaS (Shoflo, Ontime, LiveEdit
  €99–299/mo) own team budgets but have no waveform. Real workflow today = Reaper markers → CSV →
  cottage-industry converters (reapma3/ReaMarker/timecode.su) → console, + emailed spreadsheets.
- **Dance:** no tool combines waveform/count annotation + formation animation; market split into
  music-side (Anytune, Transcribe!, Moises) and formation-side (Choreographic, StageKeep, ArrangeUs)
  silos. No dance-studio management platform (Jackrabbit, DanceStudio-Pro, Akada, Studio Director,
  ClassBug) offers any choreography or music tooling — clean distribution whitespace.
- **Pricing model to copy:** Frame.io/Figma pattern — per-seat core, free guests, free tier on
  projects+storage not features, never per-project. Stage Write proves $9.99/mo + $249/yr team bundles
  work for choreography tooling.
- **Market size:** "dance studio software" figures unreliable (report mills disagree ~280×); defensible
  niche ≈ US$400–500M. Real but lifestyle-to-small-business scale on both sides — shape ambition/burn
  accordingly.
